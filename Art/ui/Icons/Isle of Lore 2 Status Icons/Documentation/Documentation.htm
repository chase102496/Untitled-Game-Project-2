Introduction

Isle of Lore 2: Status Icons is a pack of 136 icons which can be used as interface elements to indicate the current condition of a player character. They cover common statuses, buffs and debuffs and other states and conditions a game character could be in. They work great with my other Isle of Lore 2 packs, including Isle of Lore 2: Hex Tiles!

The icons come as png files with and without a background and also as Krita (.kra) and Photoshop (.psd) source files for your personal edits. The pack is compatible with my batch image editing tool Color Mapper, which means you can easily change the color palette of the icons to your own liking... the required Color Mapper Files are included!

If you bought the pack when it came out, you may notice some differences. First and foremost, the color palette is a bit darker and more saturated, making it compatible with the Isle of Lore 2: Hex Tiles update. The icons were completely re-drawn in a higher resolution (150% the size), too! Besides that, some new icons were added (Campfire, Anchor, Unlucky, Chicken, Disarmed, Demon, Star, Backpack) or changed (Lucky, Unconscious, Banner, Key, Closed Lock, Open Lock). Some extra icons (Crawling, Prone, Pushback, Bubbled, Running, Jumping, Praying, Meditating) were released as an extra download and are now part of the pack itself.

This documentation will go into more detail on how to find the files you want and how to change and re-export them to fit your needs.

Please keep the license in mind, which is available as a License.txt-file in the folder you downloaded. If you have questions or feedback, you can reach me through the following channels. Content Structure

The asset pack contains a folder called Sources and in it, you'll find the three folders input, output and misc. While misc contains a list of icon names ordered by their ID, the input and output folders are the ones relevant for a Color Mapper setup. If you don't use Color Mapper, you can head straight to the output-folder and use the files there, but if you do, please refer to the section Re-exporting the Icons with Color Mapper further down.

The icons come in two variants. The first variant places the icons on a white background with rounded corners. The second variant has a transparent background. Both variants have their respective folders status_icons.standard_background and status_icons.standard_transparent within the output-folder.

After we had a look at the output-folder, let's check the content structure of the input\status_icons-folder, where you'll find all the Krita (.kra) and Photoshop (.psd) files as well as image file exports (.png) of all the single layers of every icon. These files are required if you want to use Color Mapper. The Color Mapper Files *.colormapper.txt tell Color Mapper how to assemble the icons. The icons which have numbers on them are generated with the help of the Composite Files *.composite.colormapper.txt.

Re-exporting the Icons with Color Mapper

If you already used Color Mapper before, please get the update before using it with the Status Icons pack, as the update contains a new feature allowing to reference other files by file name instead of identifier when using composites; this Color Mapper update is required by the Status Icons pack.

If you want to use the icons the way they are, refer to the Sources\output-folders. But if you want to change the icons, for example replace some or all of the colors, you can use my batch image editing tool Color Mapper. It can also re-size, re-scale and re-position icons, all in one go.

Color Mapper assembling an icon from its single parts.

Color Mapper assembling an icon, but also switching out some of the colors.

To make Color Mapper work, the Isle of Lore 2: Status Icons download includes the folders Sources\input\status_icons, containing the Krita (.kra) and Photoshop (.psd) source files as well as png exports of the single layers and Color Mapper Files (.colormapper.txt) containing information on how Color Mapper is going to assemble the files.

Color Mapper isn't a part of this asset pack and instead has to be downloaded here, which allows me to release updates for Color Mapper independently of the projects it is used in. The download includes a Documentation-folder with exact instructions on how to use Color Mapper, but the workflow in general is at follows:

  1. Setup Color Mapper's configuration file so it knows where to find the Sources\input and Sources\output folders.
  2. Running Color Mapper, which will gather all the *.colormapper.txt-files in the input folder and assemble the png files (and composites) from the single layer files with colors referenced in the *.colormapper.txt-file and defined in Color Mapper's configuration file. The assembled png files will be put in the output-folder.
  3. After checking the results, the files from the output-folder are ready to use.

If you want to make changes to the icons themselves, you can open the Krita or Photoshop files, edit them and export the single layers accordingly. Make sure to keep the naming scheme status_icon_(name)_(id)_(layer).png intact, for example status_icon_strength_0_line.png for the line-layer of the Strength icon which has the id 0.

Some of the icons have numbers on them. Those are generated with the help of Color Mapper's Composites feature. Take a look at status_icon_afoot0_1000.composite.colormapper.txt:

width:768 height:768
file:"afoot_0.png" x:0 y:0 flipped:false
id:"status_icons\status_icon_afoot_88" x:0 y:0 flipped:false

This file tells Color Mapper to generate a new image in the size of 768x768 pixels and render, from bottom to top, the Afoot icon status_icon_afoot_88 by its id (id:) and then afterwards the number 0 icon from the file afoot_0.png by its file name (file:).
